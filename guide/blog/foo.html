搭建VuePress博客的经过与要点

# 简介

VuePress由两部分组成：一部分是支持用 Vue 开发主题的极简静态网站生成器，另一个部分是为书写技术文档而优化的默认主题。它的诞生初衷是为了支持 Vue 及其子项目的文档需求。
它以 Markdown 为中心的项目结构，以最少的配置帮助你专注于写作；享受 Vue + webpack 的开发体验，可以在 Markdown 中使用 Vue 组件，又可以使用 Vue 来开发自定义主题；它会为每个页面预渲染生成静态的 HTML，同时，每个页面被加载的时候，将作为 SPA 运行。

# 项目初始化

安装node.js,且确保你的 Node.js 版本 >= 8。

# 全局安装

# 安装
yarn global add vuepress # 或者：npm install -g vuepress
# 创建项目目录
mkdir vuepress-starter && cd vuepress-starter
# 新建一个 markdown 文件
echo '# Hello VuePress!' > README.md
# 开始写作
vuepress dev .
# 构建静态文件
vuepress build .

# 当前项目安装

使用npm init -y 去初始化一个node项目再去配置，也可以使用在你自己的node项目，推荐使用这种方法

# 将 VuePress 作为一个本地依赖安装
yarn add -D vuepress # 或者：npm install -D vuepress
# 新建一个 docs 文件夹
mkdir docs
# 新建一个 markdown 文件
echo '# Hello VuePress!' > docs/README.md
# 开始写作
npx vuepress dev docs

# npm与yarn指令对比

NPM YARN
npm init yarn init
npm install react --save yarn add react
npm uninstall react --save yarn remove react
npm install react --save-dev yarn add react --dev
npm update --save yarn upgrade

# Yarn的优点？

  • 速度快 。速度快主要来自以下两个方面：

    1. 并行安装
    2. 离线模式

  • 安装版本统一

  • 更简洁的输出

  • 多注册来源处理

  • 更好的语义化

# npm

最初yarn与npm对比有较大的优势,但是在npm5.0发布后,npm进行了较大的优化

  1. 默认新增了类似yarn.lock的 package-lock.json.
  2. git 依赖支持优化
  3. 文件依赖优化

# 执行项目

在 package.json 里加一些脚本:

{
  "scripts": {
    "docs:dev": "vuepress dev docs", //名字可修改
    "docs:build": "vuepress build docs"
  }
}

项目执行

yarn docs:dev # 或者：npm run docs:dev

导出项目

yarn docs:build # 或者：npm run docs:build

默认情况下，文件将会被生成在 .vuepress/dist，当然，你也可以通过 .vuepress/config.js 中的 dest 字段来修改，生成的文件可以部署到任意的静态文件服务器上

比较关键文件（手动创建）

  • docs/.vuepress: 用于存放全局的配置、组件、静态资源等。

  • docs/.vuepress/config.js: 配置文件的入口文件，也可以是 YML 或 toml。

  • docs/.vuepress/public: 静态资源目录。

是否不满足于官网的主题，这时我们可以运用自己的学习的vue来搭建自己的主题，也可以使用别人搭建好的主题（有时间会去研究自己搭建主题）

# 使用他人的主题

我的博客主题运用的是：vuepress-theme-reco

# 快速开始

# 安装

npm install vuepress-theme-reco --save-dev
# or
yarn add vuepress-theme-reco

# 引用

// .vuepress/config.js
module.exports = {
  theme: 'reco'
}

# 博客配置

需要熟悉vuepress的主要配置config.js以及其中的主题配置themeConfig以及vuepress-theme-reco配置再去配置将会事半功倍

# 添加博客配置

// .vuepress/config.js
module.exports = {
  theme: 'reco',
  themeConfig: {
     // 博客配置
    blogConfig: {
      category: {
        location: 2,     // 在导航栏菜单中所占的位置，默认2
        text: 'Category' // 默认文案 “分类”
      },
      tag: {
        location: 3,     // 在导航栏菜单中所占的位置，默认3
        text: 'Tag'      // 默认文案 “标签”
      }
    }
  }
}

# 自动配置侧边栏

建议全局打开自动侧边栏功能

// .vuepress/config.js
module.exports = {
  themeConfig: {
    sidebar: 'auto'//在所有页面中启用自动生成侧栏
  }
}

然后通过 YAML front matter 来禁用指定页面的侧边栏

---
# 禁用侧边栏
sidebar: false
# 侧边栏深度
sidebarDepth: 2
---

# 手动配置侧边栏

// 自动形成侧边导航
    sidebar: {
      '/guide/' :[{
        title:'博客开发',
        collapsable: true,
        children: [
            'blog/foo'
        ]}
      ]
    },
